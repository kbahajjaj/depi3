country column

The country column has 11 nominal categories. Hence, binary encoding would be more apprporiate to use since it will have 4 columns (2^4 = 16) compared to 11 columns if we use one-hot encoding.

residence column

Since residence column has 6 unique nominal values. In one-Hot encoding it would need 6 columns, where in binary coding it would need only 3 (2^3 = 8). Thus it's better to use binary encoding as we did in countries.
